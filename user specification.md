## User Specifications

### Target Users
- Beginner Software Engineering students
- Users with little or basic coding knowledge

---

### User Needs
- Simple and clear explanations of concepts (e.g. HTML, JavaScript)
- Easy instructions on how to use the website
- Quick and clear feedback when answering questions
- Content that is easy to read and understand

---

### User Interaction
- Users can:
  - Click buttons to navigate between sections
  - Type answers into a text box
  - Submit answers using a button
- The system responds immediately with feedback (correct/incorrect)

---

### User Interface Expectations
- Clean and simple layout
- Clearly labelled buttons (e.g. "Start Quiz", "View Lesson")
- Easy navigation between sections
- Readable text with headings and spacing

---

### Accessibility
- Works on desktop and mobile devices
- Loads quickly in a web browser
- Easy for beginners to use

---

### User Goals
- Learn basic Software Engineering concepts
- Practice knowledge through quizzes
- Receive instant feedback to improve understanding

---

### Summary
The user expects a simple, interactive, and easy-to-use website that provides clear instructions and fast responses.
